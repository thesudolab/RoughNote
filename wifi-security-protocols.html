Wi-Fi security, 1997 → today

WEP, WPA, WPA2, WPA3

Four generations of the same job: stop a stranger sitting outside your window from reading your traffic. Each one fixed the last one's broken lock.

WEP 1997 Broken — avoid

Wired Equivalent Privacy used one static key, shared by every device, for the network's whole lifetime. Every packet was encrypted the same way, so once an attacker collected enough traffic, the key could be recovered — often in minutes.

WPA 2003 Transitional

WPA kept WEP's RC4 cipher (so old hardware could run it) but wrapped it in TKIP, which mints a fresh key for every packet. An attacker who captures one packet's key learns nothing about the next one.

WPA2 2004 Current standard

WPA2 dropped RC4 for AES, verified via CCMP, and formalized the 4-way handshake: router and device prove they both know the network password without ever sending it over the air, then derive a session key from it.

WPA3 2018 Most secure

WPA3 replaces the handshake with SAE ("Dragonfly") — a password-authenticated key exchange that resists offline dictionary attacks — and gives each session its own unique encryption key, so capturing one conversation doesn't help decrypt any other.

Which should you use?

  1. WPA3 if your router and devices support it — it closes the gaps found in every earlier protocol.
  2. WPA2 if WPA3 isn't available. It's still solid and near-universally supported.
  3. Avoid WPA and WEP entirely — both can be broken with widely available tools.
